# Keycard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a local-first, open-source Tauri desktop app plus `keycard` CLI that stores API keys encrypted in SQLite, supports quick-save from clipboard (global hotkey + tray), profiles for terminal env injection, and matches the acceptance criteria in the design spec.

**Architecture:** A Rust workspace: `keycard-core` owns paths, crypto (Argon2id → HKDF → per-secret XChaCha20-Poly1305), SQLite (WAL + busy timeout), entries and profiles. `keycard-cli` and the Tauri `src-tauri` crate depend only on `keycard-core` for all vault access. Desktop UI (HTML/JS or lightweight framework) calls Tauri commands that delegate to `keycard-core`. CLI prompts for master password on stdin (hidden) per spec §2 — no IPC sharing in v1.

**Tech Stack:** Rust 1.85+ (adjust MSRV in workspace if needed), Tauri 2, `rusqlite` (or `sqlx` with SQLite — prefer `rusqlite` for simpler sync embedding), `argon2`, `hkdf`, `chacha20poly1305` (XChaCha20-Poly1305), `serde` / `serde_json`, `directories` or `dirs` for platform paths, `rpassword` or `dialoguer` for CLI password prompt, `thiserror` for errors. Frontend: vanilla or minimal Vite + TS (YAGNI: start minimal).

**Spec:** `docs/superpowers/specs/2026-03-29-keycard-design.md`

- [ ] **Step 2: Run — FAIL**

- [ ] **Step 3: Implement `open_vault`**

- `PRAGMA journal_mode=WAL;`, `PRAGMA foreign_keys=ON;` on every connection open, `busy_timeout=5000`, create tables:
  - `meta(key PRIMARY KEY, value)` — store `schema_version`, `kdf_salt` (blob), optional argon params JSON
  - `entries(id TEXT PRIMARY KEY, provider TEXT, alias TEXT NOT NULL, tags TEXT, created_at INTEGER, nonce BLOB NOT NULL, ciphertext BLOB NOT NULL)`
  - `profiles(id TEXT PRIMARY KEY, name TEXT NOT NULL UNIQUE)`
  - `profile_env(id INTEGER PRIMARY KEY, profile_id TEXT NOT NULL, env_var TEXT NOT NULL, entry_id TEXT NOT NULL, FOREIGN KEY(profile_id) REFERENCES profiles(id), FOREIGN KEY(entry_id) REFERENCES entries(id))`
  - `app_settings(key TEXT PRIMARY KEY, value TEXT NOT NULL)` — JSON or string values for UI prefs (idle lock minutes, clear clipboard after save, shortcut string, etc.); **Task 10/11 read/write only this table** for those settings
- Optional test: inserting `profile_env` with invalid `entry_id` fails when foreign_keys enforced
