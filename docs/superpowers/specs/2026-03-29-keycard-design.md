# Keycard — 本地 API Key 管理（设计规格）

**日期:** 2026-03-29  
**状态:** 已定稿（用户整体验收 §1–§4）

## 背景与目标

用户在各 AI Agent / IDE / CLI 场景下持有大量模型与服务的 API key，分散且敏感。Keycard 是一个**本地优先、开源**的工具：在浏览器中复制 key 后可通过快捷方式入库；在终端与桌面配置场景中通过 profile 安全消费。首版不依赖浏览器扩展；扩展作为二期能力。

## §1 架构与组件

### `keycard-core`（Rust 库，同仓）

- 负责加密保管、SQLite 持久化、领域模型（条目、profile、设置）。
- **Tauri 应用与 CLI 仅通过此库访问存储**，禁止重复实现存储或加密逻辑。

### Tauri 桌面应用

- **托盘常驻**；注册**全局快捷键**。
- **「快速保存」小窗**：将剪贴板内容作为密钥值的默认来源，用户补全别名、提供商等后写入库。
- **主窗口**：检索、编辑元数据、与复制相关的安全行为（如可选复制后清空剪贴板）、会话锁定。

### CLI `keycard`

- 与桌面应用使用**同一数据目录与同一数据库文件**。
- 负责终端侧 **profile → 子进程环境变量注入**（见 §3）。

### 二期：浏览器扩展

- 仅负责将 **URL、页面标题等上下文** 传递给本机应用；**密钥不进入扩展的持久化存储**。
- 通信通道在实现前在规格/实现计划中**二选一写死**：**Native Messaging** 或 **本机 loopback + 一次性 token**，并配套威胁模型说明（何种主体可以连接、如何鉴权）。

## §2 数据与加密

### 存储

- 使用**单个 SQLite** 数据库文件；路径与文件名在文档中明确说明，便于用户备份与排错。

### 主密码与密钥派生

- 首次创建库时设置**主密码**；使用 **Argon2id** 派生密钥。
- **主密钥不以明文形式写入磁盘**。
- 支持**可配置的空闲自动锁定**（锁定后行为见下）。

### 密钥与元数据

- **每条 secret 使用独立 AEAD**（或语义等价的独立加密单元）；库内仅存密文与 nonce。
- **元数据**（提供商、别名、标签、创建时间等）可**明文**存储于 SQLite，以支持检索与展示。
- **日志与错误信息中不得包含 secret 明文。**

### CLI 与锁定会话

首版在文档与实现中**二选一明确**：

- **(a) 推荐首版：** CLI 在需要访问密钥的子命令中**提示输入主密码**（或等价解锁），实现简单、语义清晰。
- **(b)** 仅当桌面应用已解锁时，CLI 通过**本机 IPC** 获取会话密钥（实现复杂度更高，可作为后续迭代）。

### 同步

- **首版不做云同步**。备份方式为用户自行拷贝加密后的数据库文件；若未来增加同步，需单独设计与评审（端到端加密同步等）。

## §3 入库与出库

### 入库（浏览器 copy 后）

- 通过**全局快捷键**或**托盘菜单**打开快速保存小窗。
- **密钥值默认来自剪贴板**，用户可编辑。
- **别名必填**；提供商为**预设列表 + 自定义**。
- **保存成功后是否清空剪贴板**：独立选项；**默认建议关闭**，若开启需在 UI 中给出风险提示。
- **形态检测**（如 `sk-` 前缀、`Bearer` 等）仅作 **UI 提示**，不自动改写剪贴板或密钥内容。

### 出库（终端 / 桌面消费）

- **Profile**：命名的一组映射：**环境变量名 → 库内条目 id**。
- CLI 能力（具体 flag 名称在实现前统一）：
  - `keycard run --profile <name> -- <command>`：为子进程注入对应环境变量后执行命令。
  - `eval "$(keycard env --profile <name>)"`：在当前 shell 导出变量（需在文档中说明与 shell 历史相关的风险）。
- 文档中强调：优先使用 **`run` 子进程注入**，**尽量避免**将 `export KEY=明文` 写入 shell 历史。

## §4 错误处理、安全与测试

### 错误处理

- **主密码错误**、**库损坏或 schema 版本不兼容**、**全局快捷键注册失败**等：提供用户可读的说明；CLI 以**非零退出码**表示失败。
- 任何日志、诊断输出**不得包含 secret**。

### 安全与信任

- 在文档中说明**威胁模型**：本机恶意软件、备份介质泄露、多人共用机器等场景下的风险与建议。
- 依赖版本与**可复现构建**在 README / CONTRIBUTING 中说明，便于社区审计。

### 测试

- **`keycard-core`**：单元测试覆盖加解密、读写、profile 解析等核心逻辑。
- **CLI**：在临时目录下进行集成测试。
- **全局快捷键与各操作系统差异**：发布前使用**手动测试清单**验证。

## 范围外（首版明确不做）

- 云同步与多设备自动同步。
- 浏览器扩展（二期）。
- 作为通用密码管理器替代 1Password / Bitwarden 等全功能竞品。
